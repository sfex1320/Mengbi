# 中转站请求体覆盖指南

> 本文档教你如何在「设置 → 模型方案 → 绘画模型 → 请求体覆盖（高级）」字段里填写正确的 JSON 模板，让梦笔能对接形形色色的 OpenAI 兼容中转站。
>
> 权威定义见 [`CLAUDE.md`](./CLAUDE.md) §13（请求体覆盖）与 §13a（图像模型「系列 family」）；本文档是面向使用者的操作手册。
>
> 适用范围：本字段**只对绘画模型（`type='image'`）的 OpenAI 协议 JSON body 生效**。视频模型走另一条通道（`type='video'` + `video_kind`），不读这个字段——详见第 8 节。

---

## 1. 这个字段是干嘛的（30 秒读完）

它是一块**应急补丁**，不是必填项。

- **绝大多数情况下应当留空**——默认请求体兼容标准 OpenAI 协议，能跑就别动。
- **只有当你的中转站和 OpenAI 字段习惯有出入、导致默认请求失败时**，才填这个字段把请求"修一下"再发出去。

合并语义只有 3 条规则，记住就够：

| 规则 | 含义 |
|---|---|
| 仅顶层合并 | 你写的 JSON 与默认请求体做一次 `Object.assign`，**不递归合并嵌套对象**。 |
| `null` = 删除字段 | 任何值为 `null` 的键都会被从最终请求里剥掉。这是"屏蔽默认字段"的唯一办法。 |
| `"${var}"` = 变量替换 | 若某个值**整串**写成 `"${model}"` 这种形式，会替换为运行时的真实值。仅支持整串，**不支持拼接**（`"prefix-${var}"` 不行）；变量名写错（不在变量表里）时会**跳过这一项覆盖、保留默认值**并记一条警告，不会把字段静默删掉。 |

> **它叠在「系列(family)」之上**：梦笔先按图像模型系列（family）算出默认请求体——每个 family 只发它真正识别的字段（这就是「选 4K 实际出 1K」修复的来源，见 [`CLAUDE.md`](./CLAUDE.md) §13a）。请求体覆盖是在 family 算完之后做的**最后一道顶层合并**。所以：family 决定默认字段集，你的 JSON 在它之上微调（含 `null` 删字段）。

## 2. 对账法 4 步走

接入新中转站时，按这 4 步走一遍，命中率最高。

### Step 1 — 记住梦笔的默认请求体

这是基线。每次都和它比对。对**标准 OpenAI 协议（`default` 系列）**而言，默认 body 大致长这样：

```json
{
  "model": "<你映射的实际模型 ID>",
  "prompt": "<用户输入的提示词>",
  "size": "1024x1024",
  "n": 1,
  "quality": "high",
  "response_format": "b64_json"
}
```

把这几个字段抄到草稿纸上。

> **注意**：默认请求体的具体字段集由模型的「系列(family)」决定，不是写死的——例如 `gpt-image-2` 只发 `size`+`quality`、`nano-banana-*` 只发 `image_size`+`aspect_ratio`（不发 `quality`）。你做对账时，请以**实际发出的字段**为准（family 决定默认集，覆盖在其上微调）。当前内置 5 个系列：`gpt-image-2` / `nano-banana-pro` / `nano-banana-flash` / `nano-banana-2` / `default`（兜底）。

### Step 2 — 抠出中转站文档的字段表

打开中转站的 API 文档（Apifox / Swagger / Markdown 都行），找：

- **"Body Params"** / **"请求参数"** / **"请求体"** / **"Request Body"** 这一栏
- 把所有字段名、是否必填抄下来

如果文档里没有清单只有示例 JSON，就照示例反推字段。

### Step 3 — 做一张对账表

两栏对照，逐行打分：

| 梦笔默认 | 中转站文档 | 判断 | 操作 |
|---|---|---|---|
| 有 X | 有 X | ✅ 一样 | 不动 |
| 有 X | ❌ 没 X | 🔴 左有右无 | 用 `null` 删 → `"X": null` |
| ❌ 没 Y | 有 Y | 🟡 左无右有 | 按需要加 → `"Y": "值"`（不需要就跳过） |
| 有 X | 有 Y（同一个意思） | 🟢 同义改名 | 一删一加 → `"X": null, "Y": "值"` |

### Step 4 — 把对账表翻译成 JSON

收集所有非"不动"的行，组装成一个对象，粘到设置页就完了。

## 3. 变量占位用法

如果你想让某个字段的值跟随用户在生图页选的参数走，就用变量占位。

| 占位符 | 类型 | 等价含义 |
|---|---|---|
| `${model}` | string | 实际模型 ID（来自模型映射） |
| `${prompt}` | string | 正向提示词 |
| `${size}` | string | 尺寸（如 `"1024x1024"`） |
| `${n}` | number | 生成数量 |
| `${quality}` | string \| null | 画质等级 |
| `${aspect}` | string \| null | 长宽比 |
| `${image_size}` | string \| null | 图像尺寸档位（如 `"1K"`/`"2K"`/`"4K"`） |
| `${negative_prompt}` | string \| null | 反向提示词（预留占位；`runOpenAIImage` 当前未接 negativePrompt） |

**注意事项：**

- **必须整串占位**：`"size": "${size}"` 才会被替换；`"size": "auto-${size}"` 不会。
- **null 类型变量自动剥离**：如果 `${quality}` 解析为 `null`（用户没选），剥离规则会顺手把这个字段删掉，与"`null` = 删除"语义一致。
- **写错变量名会被跳过**：占位符里的变量名不在上表（多半是拼错），梦笔会**跳过这一项覆盖、保留默认值**并记一条警告，不会把字段静默删掉。
- **不要变量套变量**：不支持。

## 4. 完整案例：私云API（shiyunapi）

走一遍真实流程，让你看到 4 步法是怎么用的。

### 4.1 文档信息

- 文档：https://shiyunapi.apifox.cn/api-448504710
- 端点：`POST https://shiyunapi.com/v1/images/generations`
- 鉴权：`Authorization: Bearer <key>`

### 4.2 私云API的字段表（Step 2 抠出）

| 字段 | 必填 |
|---|---|
| `model` | ✓ |
| `prompt` | ✓ |
| `n` | ✓ |
| `size` | ✗ |
| `quality` | ✗ |
| `format` | ✗ |

### 4.3 对账表（Step 3）

| 梦笔默认 | 私云API | 判断 | 操作 |
|---|---|---|---|
| `model` | `model` | ✅ | 不动 |
| `prompt` | `prompt` | ✅ | 不动 |
| `size` | `size` | ✅ | 不动 |
| `n` | `n` | ✅ | 不动 |
| `quality` | `quality` | ✅ | 不动 |
| `response_format` | ❌ 无 | 🔴 | `"response_format": null` |
| ❌ 无 | `format`（扩展） | 🟡 | 一般留空跳过；想要可加 |

### 4.4 最终 JSON（Step 4）

最小、最稳的版本：

```json
{
  "response_format": null
}
```

如果想顺便指定输出 png：

```json
{
  "response_format": null,
  "format": "png"
}
```

### 4.5 设置页其它字段怎么填

| 字段 | 值 |
|---|---|
| 中转站名称 | `私云API`（自取） |
| API 调用地址 | `https://shiyunapi.com/v1`（**不要**带 `/images/generations` 后缀） |
| API Key | 后台获取的 `sk-xxxxx` |
| 绘图 API 协议 | `openai` |
| 模型映射 | 显示名 `GPT-Image` → 实际 ID `gpt-image-2` |
| 请求体覆盖 | 见 4.4 |

填好后立即点"测试连通"。

## 5. 常见错误对照表（速查）

测试连通失败时，按报错文案直接查这张表。

| 报错原文片段 | 含义 | 该填什么 |
|---|---|---|
| `json: cannot unmarshal string into Go struct field` 类 HTTP 500 | new-api 衍生站对 `response_format` 字段类型不兼容 | `{ "response_format": null }` |
| `quality is required` / `field quality required` | 站点强制要这个字段 | `{ "quality": "high" }` 或 `{ "quality": "${quality}" }` |
| `unknown field: quality` / `extra field` | 站点不接受这个字段 | `{ "quality": null }` |
| 出图比例不对 / 400 不识别 `size` | 站点用 width/height 双字段 | `{ "size": null, "width": 1024, "height": 1024 }` |
| 站点要 `aspect_ratio` 而非 `size` | 字段名差异 | `{ "size": null, "aspect_ratio": "${aspect}"  }` |
| 想加水印开关 / 风格预设等 | 站点扩展字段 | 直接加：`{ "enable_watermark": false }` |
| **响应解析失败 / 没图返回 / 提示"未识别响应"** | **本字段救不了**，是适配器层问题 | 改用其它"绘图 API 协议"下拉项，或反馈让仓库新增 adapter |

## 6. 试错胜过死磕文档

文档表只是**预测**，不是真理。许多中转站文档与实现存在以下落差：

- 写了字段，但后端没实现
- 没写字段，但实际兼容
- 写了类型 A，实际接受类型 B

所以最终判断标准始终是：

> **保存配置 → 点"测试连通" → 看真实报错 → 按第 5 节速查表改一行 → 再测**

迭代两三轮通常就能定下来。比花一小时啃文档管用。

## 7. 这块字段管不到的事（边界）

为避免走弯路，明确列出本模板**做不到**的事：

| 想做的事 | 本模板能做？ | 替代办法 |
|---|---|---|
| 改 `Authorization` 头 / 加自定义鉴权头 | ❌ 仅覆盖 body | 联系仓库加新配置列 |
| 改请求路径（如 `/v2/draw` 替代 `/v1/images/generations`） | ❌ | 同上 |
| 改响应解析（如把 chat-style `choices[].message.content` 转成 `data[].b64_json`） | ❌ | 仓库新增 adapter |
| 影响"图生图"（`runOpenAIImageEdit`，FormData 上传） | ❌ | FormData 没有 JSON body 可覆盖 |
| 影响 grsai 协议（`runGrsaiImage`，异步轮询） | ❌ | 选 `grsai` 协议本身就走另一条代码路径 |
| 影响 `image_kind='comfyui'`（一键直跑工作流） | ❌ | ComfyUI 走 workflow 占位符替换（`{{prompt}}` 等），不是 body 覆盖 |
| 影响视频模型（`type='video'`） | ❌ | 视频是另一条通道，见下方第 8 节 |
| 影响对话 / 图片反推等接口 | ❌ | 本字段**只**对绘图模型生效 |

需要超出本字段能力范围时，请反馈到仓库新增配置项或 adapter，不要试图通过任何"奇技淫巧"绕过去。

## 8. 视频模型是另一条通道（别填到这里）

视频生成（2026-06-07 接入）走的是**异步范式**：提交任务 → 轮询状态 → 下载 mp4 落盘 → 入图库，与图像的同步出图完全不同。所以**视频模型不读本字段**。

- 配置：模型类型选「视频」（`type='video'`），并选「视频 API 协议」`video_kind`，可选值：`kling` / `sora` / `unified` / `seedance` / `custom` / `veo` / `runway` / `fal`。
- 其中 `kling` / `sora` / `unified` 走内置 legacy 引擎（`electron/ipc/video.ts`）；`seedance` / `custom` / `veo` / `runway` / `fal` 走供应商适配器（`electron/services/video/`，`VideoProviderAdapter` 接口 + registry）。
- 视频模型的端点、能力（capabilities）、限制（limits）、默认参数、费用阈值统一在「设置 → 视频模型配置中心」里可视化编辑（存 settings 表 `video_providers_json`），不靠 JSON 请求体覆盖。
- 凭证仍复用 `api_configs`（`type='video'`），API Key 同样经 `safeStorage` 加密。

如果你的目标是接视频中转站，请到视频模型配置那一套去配，本指南只管图像。
